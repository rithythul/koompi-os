Implementation Stack

Stack

Two toolchains, one shell language, one model runtime — decided, not vibed. QML/Quickshell draws it, Rust daemons think and guard it (ADR-0004: chosen for ecosystem — zbus, rustls, rusqlite + sqlite-vec, Automerge), Zig stays where it already works (installer, restore), and llama.cpp runs signed, locally-stored weights (ADR-0014). This is the wiring map — real binaries, real org.koompi.* D-Bus names — and where a number is unknown it says so instead of inventing one: resource budgets live in architecture.md §11, measured, not promised.

Quickshell ↔ daemons
D-Bus on the session bus (org.koompi.ContextEngine, org.koompi.Assistant1) plus busd.sock (NDJSON-RPC) for high-rate streams. QML binds reactively — no polling, no engine in the shell.
Daemons ↔ model
HTTP to localhost only. llama-server serves chat + embeddings from the signed read-only weight store (ADR-0014); it has no registry, no updater, no route off-box.
Daemons → world
Two doors, both consented, both ledgered: the cloud-LLM request and koompi-syncd. v1: the AI-plane units carry IPAddressDeny=any (kernel-enforced); P-2 (1.x) extends containment per-app via netns + nftables.

Two toolchains, by decision — not monoculture, not sprawl KOOMPI

QML for pixels, Rust for the daemons that think and guard, Zig where it already works. ADR-0004 names the reason honestly: Rust won on ecosystem — zbus, rustls, rusqlite, Automerge — not on language ideology. The boundaries are protocols (D-Bus, sockets, localhost HTTP), not shared memory, so any layer can be rewritten without touching the others.

Privacy is enforced by topology

The AI plane has two doors out — the consented cloud-LLM request and koompi-syncd — and v1 makes that kernel-true with IPAddressDeny on every other unit. A session-bus daemon that components merely consult is advice, not enforcement (arch §9); the wiring diagram, not a promise, is what keeps bytes home.

The AI runs on your silicon

llama-server loads signed weights from a read-only store and binds localhost. No registry pulls, no silent updates, no per-user copies bloating a shared lab machine (ADR-0014). The default works fully offline, in a village with no fibre, exactly as it does anywhere else — and honest tiering (§43) says what each machine can actually run.

Built to be rebuilt in 2046

Plain-text QML, reproducible cargo builds, static Zig binaries, GGUF weights in documented packages — bootstrappable from source with no proprietary toolchain. LIFETIME OS: a KOOMPI machine compiles its own shell two decades from now because nothing here depends on a vendor staying alive.

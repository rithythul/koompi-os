Implementation Stack

Stack

Every screen in this brainstorm resolves to four languages on one machine. QML/Quickshell draws it, Zig daemons feed it, a Python layer thinks for it, and Rust guards the bytes that must never leak. This is the engineering contract behind LOCAL-FIRST and PRIVACY-FIRST: the AI runs on your silicon, the daemons hold your context in RAM you own, and the only path off-device is one Rust egress gate you can read. This section is the wiring map — real binaries, real com.koompi.* D-Bus names, real socket paths — not architecture poetry.

Right tool, right layer — no monoculture KOOMPI

QML for pixels, Zig for daemons, Python for thought, Rust for secrets. Each language sits where its strengths are non-negotiable: QML hot-reloads so the shell stays iterable, Zig stays small enough to audit, Python rides the entire ML ecosystem, and Rust guarantees memory safety on the crypto path. The boundaries are protocols (D-Bus, UDS, HTTP), not shared memory — so any layer can be rewritten without touching the others.

Privacy is enforced by topology

The Python AI layer has no network capability of its own — its only route off-device is a Unix socket to koompi-egress, which consent-checks and ledgers every byte (§28). PRIVACY-FIRST isn't a setting you trust; it's the shape of the wiring diagram. There is no second path.

The AI runs on your silicon

The bridge binds 127.0.0.1:8731 and the model loads via ollama on your own GPU/CPU. LOCAL-FIRST means inference, memory, and retrieval all happen on the machine you own — the default works fully offline, in a village with no fibre, exactly as it does in a datacenter.

Built to be rebuilt in 2046

Static Zig binaries, plain-text QML, a uv-locked venv, and reproducible cargo builds — the whole stack is bootstrappable from source with no proprietary toolchain. LIFETIME OS: a KOOMPI machine compiles its own shell two decades from now, on hardware not yet designed, because nothing here depends on a vendor staying alive.

The new machine is introduced, inherits everything, and the old one is honored and verifiably laid to rest.

A ceremony, not a migration tool KOOMPI

Other OSes treat a new laptop as a setup wizard. KOOMPI treats it as succession: the elder machine introduces the successor over local mesh, hands over memory, keys, and trust postures, and is honored for its service before it's wiped.

Identity survives the architecture

x86 to ARM to RISC-V to whatever 2060 brings — every memory is tagged with the era that made it (embed_model_id), so recall stays era-correct across silicon generations. The self is the data plus the keys, never the chip.

The wipe is provable

koompi-restore destroys keys in hardware, crypto-erases the volume, verifies the surface, and issues a receipt signed by the successor. An old KOOMPI machine re-enters the world clean — by proof, not by promise.

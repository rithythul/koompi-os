The Exit Door

Futures v1

This is the canonical spec for the Lifeboat export engine. The engine ships at v1 because factory reset depends on it — reset seals a Lifeboat archive before it wipes, so the engine's completeness verifier is load-bearing from day one. On top of the engine sit four presets (snapshot §30 · legacy §24/§62 · refugee §70 · exit §81), each landing at 1.x. The exit preset (1.x) is the full-fidelity export off the platform: every store the OS registers, in a self-describing open format, with a completeness checklist that fails closed if anything is withheld.

✦ Why the engine is v1, not later
Factory reset seals a Lifeboat archive before it wipes the disk, so the engine and its completeness verifier must exist at v1 — a reset that could not prove it saved everything first is not a reset anyone should trust. Export is not a farewell feature bolted on late; it is the safety mechanism the destructive path depends on.

A format that outlives the company

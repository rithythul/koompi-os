This is the engine behind BEAUTIFUL BY DEFAULT, and it is where §01's deliberately-undefined light theme finally gets its values. Beauty here is generated, contrast-checked, and reproducible — never hand-painted per release, never accidental.

Beauty is generated, not painted KOOMPI

macOS and Windows 11 ship a fixed palette plus a handful of accent dots; GNOME bolts on libadwaita variants. KOOMPI derives the entire token set from your wallpaper through one CAM16 tonal algorithm. Every surface is coherent and personal from any image — BEAUTIFUL BY DEFAULT as an engine, not a mood board curated per release.

Light theme is no longer "TBD"

§01 deliberately left the light token values undefined because hand-authoring a second theme never scales. This engine resolves that audit gap structurally: light is the same palette read from the high end of the tone scale, contrast-checked by the same guardrail. One source of truth, two surfaces — and the gap closes for every future wallpaper, not just today's.

Contrast is a hard gate, not a hint

A generated palette that ships an illegible UI is a bug, not a style. Every foreground/background pair is checked against WCAG AA before theme.json is written; failing pairs are tone-nudged automatically, preserving hue. The OS literally cannot apply a theme it can't read.

A theme that ages with you

Themes are 2.4 KB of seeds, locks, and font pairing — signed by your Selendra DID, snapshotted by koompi-restore. In 20 years the JSON re-derives a coherent palette on hardware that doesn't exist yet, on color spaces not yet shipped. LIFETIME OS: your taste is portable data you own, never a baked asset that rots.

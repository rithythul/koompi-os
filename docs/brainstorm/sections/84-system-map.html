✦ One Ask — the session protocol
Every ask surface opens the same assistantd session: one conversation store, one memory, one consent state. A surface contributes only its context attachment — the Launcher attaches nothing, Files attaches the selected file, the Browser attaches the page, the Terminal attaches cwd + last exit code. Ask in Files, continue in the sidebar: same thread. The ✦ ASK MODE pill is the universal marker; Esc always returns to the surface.

⚖ One Policy Engine — org.koompi.Policy
A single store answers "may X do Y on Z?" for every subsystem. Tool Approval writes session policies; App Permissions and network rules write app policies; the Household Constitution is the human-readable law layer; the admin LOCK FLOOR is the tier no user action can lower. Precedence: floor > constitution > app > session. Six features become one daemon plus views — and the Egress Ledger logs against one rule ID, not five.

Consolidation is the thesis KOOMPI

Six ask boxes with six histories is the silo disease this OS exists to cure — caught growing inside our own document. One partner, one memory, one door, one law: the System Map is where that sentence becomes enforceable architecture.

Fewer surfaces, deeper trust

A user who finds the same answer in the sidebar that they started in Files learns the partner is one mind. A user who finds five permission panels learns to trust none of them. Every combine here removes a place for understanding to fracture.

The map is maintained

New sections must check in here: reuse an engine or justify a new one. The ⊕ chips on absorbed sections point back to this map, so the document — like the OS — stays legible as it grows.

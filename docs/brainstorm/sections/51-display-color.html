Edge case — display vanished mid-session
HDMI-A-1 stopped responding (cable yank or KVM switch). Its 3 windows did not get destroyed — they migrated to DP-1 and KOOMPI parked the workspace binding. When the same EDID returns, windows snap back to their original output and rotation, automatically. A dead monitor is never a reason to lose work.
Color is a functional guarantee §53

Beautiful by default is not subjective here. Every generated palette is gamut-clamped to the live ICC profile and verified against the active daltonize transform, so "emerald = safe" and "amber = consent" hold their meaning on a calibrated 4K, a cheap TN panel, and for color-blind users alike — measurably, ΔE-checked.

SDR-in-HDR done right

macOS and Windows let SDR UI ride HDR peak brightness, scorching white menus at 600+ nits. KOOMPI tone-maps SDR to ~200 nits inside the HDR signal — comfortable for aging eyes, the core of a Lifetime OS. The full range is reserved for video and games that actually authored for it.

Bound by EDID, not connector

Scaling, ICC, rotation, and workspace bindings key off each panel's EDID serial — not the connector index. Swap a DisplayPort cable, dock and undock, change a KVM input: your color and layout follow the physical screen, not the port. A detail that only bites you after ten years of hardware churn.

Location curve never leaves the device

Night Light's sunset ramp is computed on-device from latitude alone — no geolocation API, no network call. Data sovereignty extends down to the color temperature of your screen at 8pm. Auto-comfort without auto-surveillance.

Khmer is the crispness benchmark

Fractional scaling is validated against stacked Khmer glyphs, not Latin text. Sub-pixel softness English shrugs off makes ក ខ គ unreadable. Khmer-native means the hard rendering case is the default test, so every script the OS shows stays sharp.

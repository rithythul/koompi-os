Duress decoys · refused ✕ security theater
An earlier draft designed a silent decoy profile here. It is cut, honestly: a second LUKS keyslot and a decoy subvolume are visible to any competent forensic examination — fake deniability is worse than none, and a discovered decoy can convert into an obstruction charge. KOOMPI does not ship protections that only work if the adversary is careless.

Each user is a fully separate vault — own Selendra DID, own TPM-sealed key, own LUKS keyslot, own AI memory (assistantd's per-user store). Switching users unmounts the previous vault; there is no shared decryptable state. Ties to §44 profiles.
Local, key-sovereign auth Local-first

No cloud auth, ever. Biometrics match on-device against templates that never leave the secure enclave; the Selendra DID is signed inside the TPM via challenge-response. The key never enters RAM — you prove who you are without exposing what proves it.

ext-session-lock-v1, not an X11 grab

The locker uses Wayland's ext-session-lock-v1: the compositor guarantees desktop content is unreadable even if the locker process crashes. No screenshot, no overlay race, no "Xorg locker froze and exposed the desktop" — a whole bug class deleted by protocol.

Khmer-first, register-aware

Khmer numerals on the clock (០៨:៤២), Khmer day/month names, and a greeting in the formal register — because a machine addressing its owner is a formal context. Not translated chrome bolted onto English; the lock screen was authored in Khmer.

The AI cannot let itself in

Every privilege escalation routes through polkit with the AI named as requester. The agent can ask; only a human biometric or passphrase grants. Each grant is logged, attributable, and reversible — the consent gate the rest of the OS leans on.

Theater refused — no duress decoys

A decoy that forensics can find is a trap dressed as a feature: it teaches a user to rely on deniability that evaporates exactly when it matters, and its discovery can make things legally worse. The honest answers are boring and real — LUKS at rest, lockout throttling, a guest profile that holds nothing. Honesty over capability, even here.

Ages across a lifetime

Templates re-enroll as fingers age; the DID survives hardware migration via sealed backup; lockout throttles but never wipes. In 20 years the face model is retrained, the key is re-sealed to new firmware — the identity persists, the screen stays beautiful.

Motion Language

Kinetics

The OS-level motion specification the design-system audit found missing. Durations, easing curves, and a single animation grammar every panel obeys — defined once, so the whole OS feels like one calm, intentional object instead of forty surfaces each inventing their own physics. Motion here is not decoration: it tells you what the system is doing, where things went, and whether it is the machine or the AI that is busy. Includes the reduced-motion contract with §38, honored at the compositor, not faked per app.

Two engines, one vocabulary
Every motion token below ships as two implementations from a single source of truth. The shell panels (launcher, sidebar, OSD) animate in Quickshell/QML; windows, workspaces and special-workspaces animate in the Hyprland compositor. The token defines both so a single theme — or a single reduced-motion toggle — drives the entire screen.

Duration Scale
Four steps. Nothing in the OS uses an off-scale duration. Hyprland speed is expressed in deciseconds (1 = 100 ms); the token carries both.
instant
0 ms
State that must read as a fact, not an event — focus rings, checkbox ticks, key repeat.
quick
120 ms
Hovers, button presses, tooltip fade, OSD value bumps. Felt, not watched.
standard
240 ms
The default. Launcher open, sidebar slide, notification arrive, panel enter/exit.
deliberate
400 ms
Shared-element handoffs and overview zoom — long enough that the eye tracks where a thing went.

Shared-Element Handoff Grammar

Reduced-Motion Contract · with §38 Accessibility opacity-only · motion-safe · swapped at compositor
Reduced motion is a first-class curve set, not a degradation. One toggle in §38 swaps Hyprland's bezier table at the compositor — every app honors it at once, none re-implements it. Each named curve below has a defined reduced-motion sibling.
Full-motion Reduced-motion sibling
entrance (translate + scale + fade)opacity fade only · quick
spring (overshoot)no overshoot → plain entrance fade
Workspace slideinstant cut — 0 ms, no traversal

Magnifier edge-pan (§38)stays smoothed — an instant snap here is itself nauseating
The magnifier exception is the proof the contract is real and not a slogan: "reduce motion" does not mean "zero motion everywhere," it means "no motion that costs the user." §38 owns the toggle; this section owns the curve each setting resolves to.
The Meaning of Motion — what direction and speed say
↑
Up / out from origin
Created here, lives here. Returns to its origin on exit so you know where to find it again.
→
Horizontal travel
Same level, different place. Workspaces, sibling panels — lateral, never hierarchical.
⇒
Fast = trivial, slow = consequential
quick for the reversible, deliberate for the thing you should watch.
⟳
Looping = waiting on us
Any repeating motion means "not done." When it stops, the work is genuinely finished — never a fake completion.
Specified once, obeyed everywhere Source of truth

Three curves, four durations — that is the entire vocabulary. The audit found §03, §06, §10 and §04 each animating with their own implied timing and no shared file. This section is that file: every downstream panel resolves its motion to a named token, so the OS reads as one calm object instead of forty improvising surfaces.

Motion is the honest status channel

Cyan determinate bars for system work that has a shape; indigo non-directional breathing for AI cognition that does not. The user can tell at a glance whether the machine is busy or the model is thinking — TRANSPARENT AI made literal. A looping animation always means "still working"; it never lies about completion.

A curve set that ages with the user

The token table is a versioned contract (koompi-motion v1.0): a re-theme in twenty years swaps values, never the grammar, so muscle memory survives. Reduced motion is a peer curve set, not a fallback — the install that feels right at 30 still feels right at 80, and §38 flips one toggle to get there.

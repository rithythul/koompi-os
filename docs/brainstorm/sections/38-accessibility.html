A11y is a build gate, not a panel KOOMPI

Every custom-painted QML widget must declare its Accessible role, name, and action, or CI fails the merge. GNOME and KDE discover a11y gaps in the field, years late; KOOMPI catches an unlabeled button before it ever ships. Beautiful by default is enforced for everyone, by the pipeline.

Assistive tech stays local

Screen-reader TTS, voice STT, and Khmer segmentation all run on-device. The most sensitive moments — what a low-vision user reads aloud, what a hands-free user dictates — never leave the machine. Accessibility is exactly where surveillance OSes leak the most, so it is exactly where data sovereignty matters most.

First Khmer-native screen reader

Spaceless Khmer breaks every mainstream reader. KOOMPI segments by orthographic syllable and voices the correct register (formal / royal) — the same first-class Khmer the AI speaks, extended to users who cannot see the screen at all. Honest accessibility means it works in the user's language, not only in English.

Whole-system motion + scale

Reduced motion swaps Hyprland's animation curves at the compositor and fractional scaling flows through wp_fractional_scale — so one toggle changes every app at once, instead of each app re-implementing (and forgetting) the setting. The proof a Lifetime OS can keep adapting for fifty years on the same install.

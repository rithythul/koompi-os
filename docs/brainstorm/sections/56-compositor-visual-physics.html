Compositor

The Hyprland layer beneath every pixel: blur, shadow, rounding, and the layer-shell render pipeline — specified with an honest GPU budget. The glassy, kinetic feel of §53 (Theme) and §54 (Motion Grammar) is only real if the compositor can draw it at 60fps on the affordable hardware KOOMPI exists to serve. Beauty that excludes is a bug.

No beauty that excludes KOOMPI

The default tier is tuned for an Intel UHD 620 — a real laptop a Cambodian student or teacher can actually afford, not a gaming rig. Glass, motion, and shadow are specified to land at 60fps there. The high tier is a bonus, never the baseline. If an effect can't ship beautifully on the affordable machine, it isn't in the design system.

Honest budget over hidden cost

macOS and Windows 11 hide compositor cost behind "your PC doesn't meet requirements." KOOMPI probes the GPU at first boot, picks a tier, and shows you the frame budget it's spending. Every effect is a line item you can read, raise, or cut — the same transparency we apply to AI, applied to pixels.

Shares the GPU with the local LLM

Because all AI inference is on-device (§43), the compositor and the model contend for one GPU. Every tier reserves half the frame budget so that when the local model is generating, the shell never stutters. Beauty and local intelligence are co-tenants, budgeted together — a constraint no cloud-AI desktop has to think about.

Ages without a hardware tax

A Lifetime OS must look right on a 2026 netbook in 2040. Tokens are resolution- and refresh-independent (time-based motion, luminance-adaptive tint), so a future 4K/240Hz panel sharpens the same design rather than demanding a rewrite — and a decade-old iGPU simply settles into the Efficient tier, still composited, never abandoned.

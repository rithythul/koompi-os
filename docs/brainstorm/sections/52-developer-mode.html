The agent obeys §11 in a terminal too KOOMPI

VS Code and Cursor grant the AI blanket terminal trust once you click "Yes, and always." KOOMPI never does. A coding agent in your shell hits the same Allow / Allow once / Deny modal as anywhere else — and writes plus git push always re-prompt, even after "remember for this session." Developer Mode widens hardware scope, not consent.

Power is scoped, visible, reversible

Hardware grants are per-project udev + cgroup allowlists that expire on their own. The local-model token can chat but cannot touch files or call tools. While dev mode is on, your Privacy Dashboard shows an amber wide-access banner — sovereignty means you always see how open the machine currently is, and close it in one click.

Khmer-native, even at the prompt

A first-class Khmer OS that can't type Khmer in its terminal is a contradiction. KOOMPI renders fcitx5 preedit correctly over the PTY — underlined composition, grapheme-cluster cursor advance, no stray COENG on backspace. The detail every other Linux terminal gets wrong, we treat as a requirement.

A Lifetime OS for builders

Your environment is an artifact, not a fragile setup. env.lock + dotfiles snapshot into your encrypted §41 backup via koompi-restore, so the exact toolbox you built in 2026 rebuilds bit-identically on a machine you buy in 2046 — and reverting dev mode hardens access without ever destroying the work.

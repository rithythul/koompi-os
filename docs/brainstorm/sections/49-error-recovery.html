Why this copy, not a translation
No jargon — no SIGSEGV, no "fatal error". The verb register is informal-warm (មិនមែនជាកំហុសរបស់អ្នកទេ — "this isn't your fault") rather than formal-cold. Khmer error strings are authored first-class, reviewed for register, never run through a translation memory. The recovery action leads; the dismiss is secondary — same hierarchy as the English dialog above. Royal/formal register is reserved for institutional deployments and toggles in §33.
Never blame the user Tone

Every failure string is audited for one rule: it states the cause, not the culprit. "Writer ran out of memory" — not "You opened too many files." macOS apologizes; Windows shows a stop code; KOOMPI explains. Blame erodes the lifetime trust the OS is built to earn.

A path back, always

No error is a dead end. App crash → restore work. Disk full → AI-ranked reclaim. Permission denied → the exact §42 grant that fixes it. Kernel panic → §30 BTRFS rollback. The recovery action is the primary button in every surface; dismissal is always secondary.

Failure stays local

Crash reports are written to disk and shown to you in full before a single byte leaves. Paths and usernames are redactable line-by-line; "Always ask me first" is honored forever. A permission-denied event is recorded in the Egress Ledger (§28), not hidden. Sovereignty holds even when things break.

Degraded, not dead

When koompi-contextd or koompi-agent-memd crash, the desktop keeps working — search falls back to filenames, the AI sidebar pauses cleanly, daemons auto-restart and re-index silently. A Principal-Engineer detail: the watchdog uses exponential backoff and surfaces the degraded banner only after the second failed restart, so a single transient crash never nags the user.

Recovery that survives decades

The pre-boot recovery shell loads its own early keymap (Khmer/English on F2) and never depends on the graphical stack it exists to repair. Boot attempt 3 auto-rolls-back to last-known-good. In year 20, after a thousand updates, the path home is still one snapshot away — the Lifetime OS promise made durable.

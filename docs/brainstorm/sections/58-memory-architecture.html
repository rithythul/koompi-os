Memory Architecture

AI v1

koompi-assistantd's memory engine (Rust, ADR-0004) is one embedded SQLite store with four namespaces: Working (in-RAM attention ring), Episodic (append-only WAL log), Semantic (knowledge graph + sqlite-vec), and Procedural (habit DAGs). One load-bearing rule governs the whole stack: forgetting is a ranking property, never a delete. Decay and pruning touch only the rebuildable retrieval index — the raw episodic record in @data is immortal by default, encrypted, and yours: only you can destroy it, through Ceremonial Forgetting, never the OS. Every vector carries embed_model_id (BGE-M3, ADR-0006), so a 2026 record stays re-readable under the 2050 embedder that doesn't exist yet. The layer names below are roles inside one Rust daemon — not separate services. v1 ships the schema, the store, and the nightly consolidation job.

Forgetting is ranking, not deletion KOOMPI

The contradiction "never delete, yet prune synapses" resolves with one rule: decay and pruning act only on the derived retrieval index — edge weights, salience, hot-vector rows. They never touch the raw episodic record in @data. A pruned memory is one the OS won't surface unprompted; given the right cue it rebuilds from the immortal substrate. Exactly human: you "forgot" a name until the photo brings it back. The one exception is the owner's alone — Ceremonial Forgetting's destroy verb, ceremony-gated and never automatic.

It survives its own substrate

Today's embedder, BGE-M3, will not exist in 2050 — and every vector is locked to the model that produced it. So the raw text must be the durable layer, precisely so the index can be re-embedded under future models. Each row carries embed_model_id: "who you were at 25" is reconstructed through era-correct vectors, never flattened by a 2050 lens. The four-layer split isn't biomimicry — it's the only structure that outlives 30 years of model churn.

Reconstruction with citations, never confabulation

Recall is constructive: the daemon blends goal, mood, temporal anchor and active attention, retrieves by vector⊕FTS5 (RRF), expands along graph edges, then narrates. Every span is tagged recorded (clickable episodic source + timestamp) or inferred (flagged "I'm reconstructing this"). An OS entrusted with a human life that confabulates is a betrayal — TRANSPARENT AI means auditable down to the timestamped row.

All four layers, local and yours

No Redis, no Neo4j, no JVM — one embedded store (sqlite-vec + FTS5 + graph tables) in the Rust daemon, sized for the 4GB/CPU-only floor. The layer names are roles, not network services. Episodic data never egresses for consolidation; sync is E2EE and opt-in; --full reset wipes the disposable index and leaves the canonical record exactly as durable as you chose. DATA SOVEREIGNTY is a control-flow guarantee, not a marketing line.

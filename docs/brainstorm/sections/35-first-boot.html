First Boot

Day Zero

Power-on to productive desktop, with nothing assumed. Every other section in this document describes an OS that already knows you — your habits, your files, your memory graph. This one designs the moment before any of that exists: the empty baseline. On day one KOOMPI knows nothing about you, and it says so honestly. Trust is earned before a single byte is indexed. Every capability is opt-in, off by default, and the very first impression is in Khmer — because who this is built for should be visible at minute zero, not buried in a locale menu.

Edge case · No network at first boot offline
A student opens KOOMPI in a province with no Wi-Fi. The model is not downloaded yet. The OS must never hold your own computer hostage to a download. Setup completes; the desktop is fully usable; only the cloud-fetched model is deferred.
✓ Shell, files, apps, settings — all work ✓ Khmer keyboard & rendering offline ⏳ AI queued — downloads on next connection
Principal-engineer detail: the ISO ships a tiny 0.5B "stub" model so semantic search and the briefing degrade-but-function from the first second offline. The 8B fetch is a background upgrade via koompi-restore's snapshot-safe queue, resumable across reboots — not a setup blocker.

Off by default is the thesis, not a setting Local First

Every index source ships disabled. The all-off state is framed as a feature — "you are private by default" — and only an explicit consent action flips a toggle to emerald. macOS and Windows opt you into telemetry and cloud sync during setup and make you hunt to leave; KOOMPI starts from zero and earns each capability.

"I don't know you yet" — honesty over theatre

This section patches the Vision audit's #1 hole: every longitudinal feature assumes a populated memory graph. Day one has none, so the empty Context Pulse says so plainly instead of faking a loading state or inventing a fake briefing. An OS that admits ignorance on day one is one you can trust on day 3,000.

The quiet choice is never the harder one

The identity fork gives "stay local-only" identical visual weight to creating a Selendra ID — no recommended badge, no dimmed skip link, no dark pattern. Data sovereignty is meaningless if the private path is the inconvenient one. And the degraded-tier model means a 4 GB laptop in a Cambodian classroom still gets a working AI — the lifetime promise has to start on the hardware people actually own.

Reversible by construction KOOMPI

Every apply is wrapped by an automatic BTRFS snapshot from the §30 koompi-restore engine and a GRUB rollback entry — created before the first byte is written. macOS and Windows hide updates behind a progress bar and a prayer; KOOMPI guarantees a working system to fall back to. A rolling release that can't break boot.

Staged, never half-applied

Packages download to a staging subvolume and activate atomically on the next boot — not file-by-file over the live system. Pull the power mid-update and you simply boot the old root. This is the detail that separates "I bricked my Arch box" from a Lifetime OS.

Risk-grouped review, not a wall of text

The changelog is sorted by blast radius: kernel and Mesa first (amber), system daemons next (cyan), user apps last (emerald). You see what can break boot before you ever see firefox's patch notes. GNOME and KDE dump an undifferentiated package list; KOOMPI tells you where the risk actually lives.

Bandwidth-aware for Cambodia

On a metered hotspot KOOMPI holds the download and offers a 12 MB security-only path. Offline, the last sync stays staged and applies with no network at all. Update cadence respects the user's connection — not Silicon Valley's assumption of unlimited fiber.

Auto-rollback on boot failure

If the new root fails to reach hyprland-session.target, koompi-restore's pre-display fallback shell offers one-click rollback and counts down to do it automatically. The user never has to know what initramfs is to recover — the OS heals itself and explains what happened in plain language.

Snapshots age gracefully Lifetime

Rollback snapshots are copy-on-write — near-zero cost at creation — and pruned after 12 generations while the human-readable changelog is kept forever. Across decades the disk never silently fills with old roots, yet the history of every change the machine ever made to itself stays auditable.

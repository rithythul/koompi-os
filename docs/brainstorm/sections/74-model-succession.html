Model Succession

Futures RULE · v1

A local model gets swapped; its embeddings should never be load-bearing. The v1 data rule: raw captured text is canonical and kept forever; vectors are a derived, era-tagged, disposable index. Every vector carries the embed_model_id that produced it (BGE-M3 at v1, per ADR-0006), so recall stays era-correct, and any embedder can be replaced by re-deriving vectors from the text that never moved. There is no "soul file" and no succession ceremony to build — just the discipline that the index is rebuildable and the source of truth is the raw text.

Canonical vs. derived — what survives an embedder swap
├─ raw text canonical · kept forever · never re-derived
├─ vectors derived · disposable · rebuilt on demand
├─ embed_model_id stamped per record (v1: BGE-M3, ADR-0006)
└─ recall era-correct: read a record with the model that indexed it

Retiring a model is a re-index, not a migration. Because nothing derived is treated as ground truth, a new embedder changes retrieval quality, never the record.

Re-embed runbook
embedder succession is gated · G-EMB-2
GATED · G-EMB-2
koompi-mem reindex --embedder <new>
├─ read raw text (canonical store, untouched)
├─ derive new vectors · stamp new embed_model_id
├─ shadow new index built alongside old · both readable
├─ eval ≥ +5% on KOOMPI-EVAL-RETRIEVAL · license gate
└─ cutover old vectors dropped only after eval passes

G-EMB-2 blocks any embedder change until a re-embed runbook exists in CI and the candidate clears the retrieval and license bars. Until it does, BGE-M3 stands. The runbook runs on the raw text, so a swap is always reversible — the index can be rebuilt from source at any later date.

Derived data is never load-bearing KOOMPI

Vectors are an index, not a record. Treating raw text as the single source of truth is what lets the embedder be replaced without touching a lifetime of memory.

Succession is a CI job, not a ceremony

Every vector's embed_model_id keeps recall era-correct, and G-EMB-2 makes replacing the embedder a gated, tested re-index against KOOMPI-EVAL-RETRIEVAL.

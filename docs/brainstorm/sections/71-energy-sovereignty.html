Energy-Sovereign Computing

Futures RULE · v1

This is an engineering rule shipping at v1 (architecture §11): heavy background work — memory consolidation, re-embedding, format transcription — runs only when three gates are all open at once: the machine is on AC power, the user is idle, and the CPU is within thermal limits. If any gate closes, the work pauses and waits; it is deferred to the next window, never offloaded to a server. On battery, KOOMPI does light capture only.

idle no contention with your foreground work; wakes on input gate 2
cool CPU package within thermal budget; throttles down before it cooks gate 3
koompi-assistantd · sleep-tier scheduler
├─ on_ac true
├─ idle true (12m)
├─ thermal within budget (61°C / 85°C)
└─ decision gates open → run T2/T3 · else defer
Deferred is not offloaded. Work that can't run waits for the next window on this machine; nothing falls back to the cloud (ADR-0001: degrade local, never silent cloud).
Consolidation is shiftable load KOOMPI

Memory consolidation is energy-hungry but never urgent — the ideal work to gate. KOOMPI runs T2/T3 sleep tiers only on AC, idle, and cool, so the heavy thinking happens when it costs the user nothing.

Scarcity never becomes surveillance

The cheapest response to a dying battery is "run it in the cloud" — the exact ratchet ADR-0001 forbids. Under the gate, deferred work waits for local power instead. Degrade local, never silent cloud.

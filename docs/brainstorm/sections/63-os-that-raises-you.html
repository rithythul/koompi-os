Futures RULE · v1

The concrete, shipping piece is the per-user policy LOCK floor from architecture §10. A guardian sets a policy floor on a managed profile — allowed capabilities, egress destinations, tool-approval strictness — and locks it. The locked profile cannot raise its own ceiling; it can only operate at or below the floor. Release is an explicit, dual-signed transfer, not an automatic birthday. This is an access-control mechanism the policy engine already enforces, not the OS "parenting" anyone.

release dual-signed transfer only (guardian ✓ + holder ✓)

The floor is a locked policy record, not a settings toggle the profile owner can flip. Attempts to widen scope below a guardian co-sign are refused at the policy gate and logged. Lifting the floor requires both signatures, which the OS records as a one-way, verifiable event.

System

KOOMPI OS can reshape itself for different contexts. Profiles switch the shell layout, AI behavior, privacy posture, and power profile in one action — changing the OS's posture for the task at hand. Naming note: "Subsystem" is reserved for the isolation system (Light / App Window / Detonation Chamber — CONTEXT.md); these are Profiles.

One atomic switch Shell

Profile switching is handled by the shell daemon — no process restarts. Bar, sidebar, AI behavior, and firewall rules all transition atomically via a single IPC message. Fast enough to feel instant; measured before promised.

AI-triggered auto-mode

The context engine can suggest or automatically trigger a mode switch. "Calendar shows 'Investor Call' in 5 minutes → switch to Presentation mode?" — with a dismiss option.

Per-mode Hyprland profiles

Each mode carries a Hyprland config delta — gap size, animation speed, workspace layout. Focus mode collapses animations; Creator mode enables precise tiling for reference windows.

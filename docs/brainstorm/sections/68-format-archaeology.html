Format Archaeology

Futures RULE

This is an engineering rule enforced now, not a far-future ambition: KOOMPI writes and exports only open, documented formats. Every format the OS emits has a published schema, and the Lifeboat export is exercised in CI — a test round-trips a real archive and asserts a system holding only the bytes, with no KOOMPI code, can read it back. Originals are never discarded; a proprietary import sits beside its open transcription, checksummed.

The rule — open formats only · schemas documented · export exercised in CI
export targets .md / .txt / .csv / .odt / PDF/A · open specs only enforced
schemas memory / egress / archive container — each versioned & published documented
proprietary imports transcribed to an open target · original kept, checksummed addition, not replacement
CI round-trip export → read back with no KOOMPI binary → assert lossless gates the build
Every artifact in @data is self-describing: bytes travel with a plain-text header naming the format, the spec version, and how to read them. If a future PR adds a closed export target or breaks the round-trip, the CI gate fails.

Same discipline already protects memories: koompi-assistantd tags every vector with its embed_model_id, so recall is era-correct. Format archaeology extends that rule from meaning to bytes.

Openness is a gate, not an aspiration KOOMPI

Many projects say they use open formats; KOOMPI proves it on every commit. The export is round-tripped in CI against a reader that has no KOOMPI code, so "your data outlives our software" is a test that must stay green, not a slogan.

Originals are sacred

Transcription is addition, never replacement. A proprietary import sits beside its open descendant forever, checksummed, in case a future reader can do better than ours did.

Shelved

The "100-year test" framing — a daemon continuously transcribing endangered formats forward for decades so a 2126 system can reconstruct a whole life — stays on the horizon shelf (horizon.md).

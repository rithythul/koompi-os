Local-only buffer

Every OS has a clipboard. Almost none treat it as the privacy-sensitive object it is — a buffer that silently holds your passwords, 2FA codes, bank numbers, and half-written messages. KOOMPI's clipboard is local-only by construction, redacts secrets before they ever land in history, and lets you scope retention per source. Super+V opens it as a companion to the §03 launcher. Edge cases a real clipboard hits
▸ Password manager auto-clear race: KeePassXC clears the clipboard after 10s. KOOMPI honors the x-kde-passwordManagerHint MIME flag and never even races to read it.
▸ Multi-flavor paste: one copy carries text/html, text/plain, and an image. KOOMPI keeps all flavors so the target app picks the best.
▸ Large image guard: a 4K paste over 12 MB stores a thumbnail + on-demand handle, not a RAM copy per history slot.
▸ Primary vs Clipboard selection: X11/Wayland's middle-click PRIMARY buffer is tracked separately and never persisted — it's transient by design.
Secrets never touch disk Privacy-first

A small Zig watcher scores every clipboard event for entropy and known token shapes (AWS keys, JWTs, private keys, OTP codes, IBANs). Matches are masked, held in locked memory for a single paste, then zeroed. Unlike macOS Universal Clipboard or Windows Cloud Clipboard, a KOOMPI secret is never written to history, never indexed, never synced.

Local-only by construction

GNOME and KDE clipboard managers are local too — but KOOMPI makes it provable. The clipboard daemon has no network capability in its sandbox profile; the egress firewall would block it even if compromised. The "0 bytes ever left" counter is read from the kernel, not a promise.

Khmer is first-class, not pasted-through

History stores and searches Khmer with correct Unicode normalization, so សៀមរាប matches whether typed with subscripts composed or decomposed. "Paste as Khmer" runs the local model with register awareness — formal, informal, or royal — so the sovereign default works in the user's own language.

Ages across decades

Ephemeral history is RAM-backed and self-expiring, so it never bloats. Only pinned snippets persist — and they live in a plain, portable store the user can read, export with their Selendra identity, or carry to a new machine 20 years from now. The clipboard that grows for a lifetime is the part you chose to keep.

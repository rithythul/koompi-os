KOOMPI Browser

AI App

The partner's window to the web — not an app with AI bolted on, and not a second brain. The browser is a citizen of the OS: it queries the one index (koompi-contextd), writes to the one memory (agent-memd), and leaves the machine only through the one door (koompi-egress). Tabs give way to task-scoped canvases; hostile pages meet the Detonation Chamber; and every AI re-rendering of the web shows its work.

A citizen, not a silo KOOMPI

Chrome ships its own sync, its own AI, its own password vault — a parallel universe on your desktop. The KOOMPI Browser refuses: one index, one memory, one consent model, one door out. Built with the OS discipline — Zig daemons, Rust on the security surfaces, Python tools the partner calls via the bus, QML chrome that matches the shell.

The web, with its work shown

Re-rendered pages carry the AI-assisted mark with the original one keystroke away; generated tables cite every cell; recorded and inferred are visually distinct. The browser never silently rewrites the web — that honesty is what makes the rewriting trustworthy.

Ships small, frames long

v1 rides the Naga release: engine + contextd ingestion + reader re-render + a handful of Tool-Approved automations. Builder Canvases, bus capabilities, and IPFS wait for 1.x. Same rule as everything else here: demoable at CODE-C 2026, still coherent in 2076.

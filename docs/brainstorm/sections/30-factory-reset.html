koompi-restore daemon Zig

The reset engine is koompi-restore, a Zig-native daemon running as a systemd service. It handles secure erasure, factory image verification, and BTRFS snapshot rollback without root shell scripts.

BTRFS snapshot rollback

If BTRFS is the filesystem, koompi-restore can roll back to a named snapshot (e.g., post-install, last-week) rather than doing a full wipe. Instant and reversible within the snapshot window.

Hold-to-confirm UX

Destructive actions require a 3-second press-and-hold, not a click. This prevents accidental resets while preserving keyboard accessibility — the hold timer is announced by a progress ring.
